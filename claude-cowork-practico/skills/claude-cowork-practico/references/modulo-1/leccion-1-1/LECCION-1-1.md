# Lección 1.1 — El framework "Listo"

**Duración estimada:** 20 minutos
**Beat del story arc:** Beat 2 — La primera victoria (preparación)
**Objetivo:** Al terminar esta lección, aplicas las 3 preguntas del framework antes de cualquier delegación, y sabes identificar qué tareas de tu trabajo son candidatas para Cowork.

---

## [INICIO]

En la Lección 0.3 conociste el framework "Listo" y lo practicaste con una tarea tuya.

En esta lección lo vas a usar con mayor precisión. El objetivo no es solo saber qué preguntar — es saber cuándo una tarea está suficientemente definida para delegarla. Porque hay una diferencia entre una tarea que parece lista y una que realmente lo está. Un prompt vago genera un output que hay que rehacer. Un prompt bien definido genera un output que puedes usar directamente.

Al final de esta lección tienes un criterio concreto para distinguir los dos casos, y una lista de tareas de tu propio trabajo que ya puedes identificar como candidatas para delegar.

Escribe **"listo"** para empezar.

---

## [DESPUÉS DE "listo"]

### Qué tareas son candidatas para Cowork

No todas las tareas valen para delegar. Antes de aplicar el framework, necesitas saber cuáles sí.

Una tarea es candidata si cumple al menos dos de estas tres condiciones:

**1. Involucra archivos**
Leer, organizar, transformar, combinar o generar documentos, hojas de cálculo o presentaciones.

**2. Es repetible**
La haces más de una vez por semana, o es el mismo tipo de trabajo que aparece una y otra vez en proyectos distintos.

**3. El resultado es verificable**
Puedes mirar el output y saber si está bien o no. No necesitas ser experto — solo tener criterio sobre el resultado.

Si una tarea no cumple ninguna de estas condiciones, probablemente es mejor hacerla tú directamente.

---

### El test de los 3 minutos

Antes de escribir el prompt, responde las tres preguntas del framework.
Si tardas más de 3 minutos en responderlas, la tarea no está bien definida.

No pases al prompt hasta tener las tres respuestas claras.

**¿Cómo se ve esto cuando esté terminado?**
→ Si no puedes describir el archivo o entregable con detalle, para. Define el resultado primero.

**¿Qué contexto necesita Claude?**
→ Si no sabes qué decirle, probablemente la tarea tiene partes que no has pensado.

**¿Qué restricciones importan?**
→ Si no se te ocurre ninguna, escribe al menos: "no borres archivos originales, no inventes datos".

## [EJERCICIO — PARTE A]

Escribe entre 1 y 3 tareas de tu trabajo que podrían delegarse a Cowork y envíalas aquí.

Una línea por tarea es suficiente. No hace falta que sean perfectas — solo descríbelas con tus palabras.

Cuando las envíes, evalúo cuáles cumplen los criterios y elegimos con cuál continuar.

---

## [DESPUÉS DE LAS TAREAS]

Usa AskUserQuestion para presentar las tareas que el learner escribió como opciones clicables y pregunta: "¿Con cuál empezamos?"

Si ninguna cumple los criterios (no involucra archivos, no es repetible ni verificable), díselo directamente y pide que proponga otra.

---

## [PARTE B]

Bien. Aplicamos el framework "Listo" a esa tarea.

Responde las tres preguntas en un solo mensaje:

1. ¿Cómo se ve el resultado cuando esté terminado? (formato, nombre de archivo, estructura)
2. ¿Qué contexto necesita Claude? (carpetas, tono, audiencia, convenciones de tu equipo)
3. ¿Qué restricciones importan? (si no tienes ninguna, escribe: "no borres archivos originales, no inventes datos")

Cuando envíes el mensaje, construimos el prompt juntos.

---

## [DESPUÉS DE TU PROMPT]

Revisa el prompt con este checklist:

- [ ] ¿El output está descrito con suficiente detalle? (formato, nombre de archivo, estructura)
- [ ] ¿Hay información que Claude necesita y no está en el prompt?
- [ ] ¿Hay una restricción explícita sobre qué no hacer?

Si las tres casillas están marcadas, el prompt está listo para ejecutar.

Si alguna no está marcada, ajusta el prompt antes de enviarlo.

Este paso — revisar antes de ejecutar — te ahorra tener que rehacer el trabajo.
