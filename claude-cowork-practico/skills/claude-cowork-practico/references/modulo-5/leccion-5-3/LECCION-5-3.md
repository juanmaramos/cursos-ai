# Lección 5.3 — Cowork para cualquier rol

**Duración estimada:** 25 minutos
**Beat del story arc:** Beat 5 — El rol amplificado
**Objetivo:** Al terminar esta lección, has identificado 3 tareas semanales de tu rol que puedes delegar desde esta semana y has generado el primer output de al menos una de ellas.

---

## [INICIO]

Esta es la última lección del curso.

Si no eres PM de producto ni founder, esta lección es para ti — independientemente de tu rol. Marketing, ops, finanzas, RRHH, recruiting, administración, consultoría, diseño. El patrón es el mismo: identificas tus tareas recurrentes con archivos y las delegas.

El objetivo no es darte un flujo genérico. Es que identifiques las tres tareas más concretas de tu semana que puedes delegar a Cowork a partir de hoy, y salgas de esta lección con al menos una ejecutada.

Escribe **"listo"** para empezar.

---

## [DESPUÉS DE "listo"]

### Primero: identifica tus tres tareas

La carpeta `modulo-5/leccion-5-3/practica/` tiene el archivo `tareas-semanales-marketing.txt`.

Es una lista de todas las tareas que hace un equipo de marketing cada semana: lo que ocurre cada lunes, cada martes, mensualmente.

Si tu rol es marketing, úsala como referencia. Si tu rol es ops o finanzas, piensa en el equivalente para tu área — el cierre mensual, los reportes de equipo, los informes de presupuesto.

→ Copia este prompt y envíalo aquí mismo:

```
Lee el archivo modulo-5/leccion-5-3/practica/tareas-semanales-marketing.txt.

Identifica las tareas de esa lista que tienen estas características:
- Involucran archivos o datos
- El output es predecible (sabes cómo se ve cuando está bien)
- Se repiten cada semana o cada mes

Para cada tarea identificada:
- Describe el prompt que usaría para delegarla a Cowork
- Estima el tiempo que tarda manualmente
- Indica si se puede hacer en batching con otras

Guarda el resultado como MIS-TAREAS-DELEGABLES.md.

Nota: si tu rol es diferente al de marketing, adapto la lista a tu área. Solo dime cuál es.
```

Vuelve cuando Cowork haya generado el archivo.

## [DESPUÉS DEL ANÁLISIS]

Abre `MIS-TAREAS-DELEGABLES.md`.

Elige la tarea que más tiempo te consume y que tiene el prompt más claro. Esa es con la que empezamos.

Escribe el nombre de la tarea que elegiste para continuar.

---

## [DESPUÉS DE ELEGIR LA TAREA]

Usa AskUserQuestion con esta pregunta y estas dos opciones:

- Pregunta: "¿Tienes ya los archivos de input para ejecutar esta tarea?"
- Opción A: "Sí, tengo los archivos"
- Opción B: "No tengo archivos, prefiero un ejemplo"

---

## [SI ELIGE "Sí, tengo los archivos"]

Dame dos cosas:

1. ¿Qué archivos o inputs tienes disponibles?
   (nombres, tipo de contenido, formato)

2. ¿Cómo se ve el output cuando está bien hecho?
   (un Excel con X columnas, un Word con X secciones)

Con eso construimos el prompt exacto para tu tarea.

---

## [SI ELIGE "No tengo archivos, prefiero un ejemplo"]

Sin problema. Vamos a usar una tarea de ejemplo para que practiques el patrón. Cuando tengas tus archivos reales, aplicas exactamente lo mismo.

> Esta lección cubre tres roles distintos — marketing, ops y finanzas — y no todos los learners tienen sus archivos de trabajo disponibles en este momento. El objetivo no es completar tu tarea real ahora, sino que salgas sabiendo exactamente cómo ejecutarla cuando llegue el momento.

La tarea de ejemplo: tienes la estructura de trabajo semanal de un equipo de marketing. Necesitas generar el informe semanal para dirección — con estado de cada área, tareas completadas y pendientes para la semana siguiente.

Es el patrón más común en marketing, ops, RRHH o finanzas. Si tu rol es diferente, el mecanismo es idéntico.

→ Copia este prompt y envíalo aquí mismo:

```
Lee el archivo modulo-5/leccion-5-3/practica/tareas-semanales-marketing.txt.

Genera INFORME-SEMANAL-MARKETING-EJEMPLO.md con esta estructura:

1. Resumen ejecutivo (3-4 líneas con el estado general de la semana)
2. Estado por área:
   - Redes sociales
   - Campañas de ads
   - Newsletter
   - Coordinación con ventas
3. Tareas completadas esta semana
4. Pendientes y próximos pasos para la semana siguiente

Tono: ejecutivo, directo, para dirección.
Donde no haya datos concretos en el archivo, escribe "[dato pendiente]" — no inventes métricas.
```

## [DESPUÉS DE LA EJECUCIÓN — aplica a ambos casos]

Revisa el output con el checklist habitual:
- ¿Tiene la estructura que esperabas?
- ¿Los datos son correctos?
- ¿El tono es adecuado para el destinatario?

Si necesita ajuste, corrígelo en una sola iteración.

Cuando el output esté listo, escribe **"listo"**.
